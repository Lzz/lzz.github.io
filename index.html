为Dell R720 安装 ssd 做系统盘遇到的问题

发表于 2016-09-15   |   分类于 编程

问题1. Dell R720 是否支持ssd
google了一下，有说支持的，有说不支持的，不管了，先买一块再说（intel 535 sata 2.5）,回来安装才发现没有合适的托架（心好凉），来回对比了一会儿发现直接安装在r720提供的托架上即可（总算是装上了），然后迫不及待的开机，发现可以识别（开心）。

问题2. 要不要给ssd做raid0
又是一顿搜索，有说做好的，也有说做了反而会降低性能的，想了一下还是先不要做试试，于是直接在ssd上安装了windows server 2008 r2,结果进系统后程序各种崩溃（连系统程序都崩溃），就怀疑是ssd的质量有问题，于是下载hd tune测试，结果发现了好多的错误于是就联系卖家，结果卖家说他们只承认intel官方的测试工具，那我就下载官方工具来测试，官方的测试工具确实没有测也问题。想了想不会是接口有问题吧，于是换了一个插槽，再用hd tune测试，果然没有错误了。于是满心欢喜的开始部署程序，结果要是一顿崩溃（泪崩）。无奈的我想到会不会是系统有问题，然后我装了3个不同地方下载的系统（全是windows server 2008 r2），结果…………还是不行啊。最后没办法使出杀手锏，给dell售后服务打电话，经过一顿的沟通dell售后告诉我这个服务器只支持pcie的ssd(fuck…..)。好吧，我就换pcie，我正要在jd上买pcie的ssd的时候dell的售后打过电话来跟我说既然我能安装上操作系统应该也能兼容sata的ssd(汗啊）,让我给ssd做个raid0试试，他要不说我都差点忘了，还没试过做raid0，于是操起光盘，插上电源开干。再次开机后一切正常！！！

ubuntu下sslocal+SwitchyOmega的配置问题

发表于 2016-09-05   |   分类于 编程

在xubuntu上安装了sslocal，chrome中也安装了SwitchyOmega，然后写好shadowsocks的配置文件启动sslocal，提示一切正常，可就是在chrome中无法连接到shadowsocks，搞了一个多星期也不知道怎么会事，最后没办法下载了一个ShadowSocksQt的gui程序，看到上面有两个选项socket5和http(s)，知道原因的我眼泪流下来！
原因是我的SwitchyOmega中的代理协议默认是http(s),而sslocal用的是socket5协议，把SwitchyOmega中的协议类型换成Soket5后我又看到了熟悉的Google。
